A New Personal Website Project

Michael starts building a fully customizable and extensible website with Twitter Bootstrap and Django on Amazon EC2 cloud servers.

First content will be migrated from PortfolioBox and WordPress. But it takes time.

Progress Log

  • May 10, 2015

    • Purchased the dedicated domain name yanshuhong.com.
    • Purchased Hatchling Plan at HostGator for web application hosting.
    • Set up Django as the server-side framework. (The power of Django definitely exceeds the needs of a personal website, but who knows what this website will be evolved into.)
    • Created new repo on GitHub and its local clone.
    • Linked against Twitter Bootstrap to enhance user interface and front-end development.
    • Set up server-side environment and fCGIs to redirect url requrests to Django.
    • Posted the front page and the new personal website is acutally running!.

  • May 11, 2015

    • Populated the navigation bar structure. Revised base.html to dynamically display the selected navbar item.
    • The second page idea page is up. It needs to be tested on the server on URL redirection.

    • Migrated 3 essays from original Wordpress.

    • Installed Google font and changed the base font to Oxygen.

  • May 12, 2015

    • Set the maximum width of the text container of essay page.
    • Migrated server-side database from Sqlite to MySQL and resolved the admin OperationalError.
    • HostGator does not have GCC to install a dependency for MySQL. Decided to switch server.
    • Issued a termination for HostGator package and registered an account at Amazon AWS.
    • Launched a EC2 cloud server and configured environment for django depolyment.
    • Enabled UTF-8 support on MySQL database.
    • Migrated project onto AWS server at ec2-52-24-225-153.us-west-2.compute.amazonaws.com.

  • May 15, 2015

    • Installed local MySQL database server.
    • Because a new domain name can not be transfered within 60 days from its initial purchase, I purchased another domain name hongyanshu.com on GoDaddy.com.
    • Associated an elastic IP to the EC2 instance and configured the domain DNS to resolute to the elastic IP.
    • Added all related IPs and hostnames to django setting to fix the Bad Request issue.

  • May 16, 2015

    • Updated base font family to be the order: Oxygen, ヒラギノ角ゴ Pro W3, Osaka, メイリオ, ＭＳ Ｐゴシック, sans-serif
    • Integrated Font-Awesome into the project.
    • Added optional preface for blog posts.
    • Implemented a basic HTML render engine for essays. [IMG *= 'name' 'width'] inserts an image tag with automatic reference to the corresponding file in the static file folder. [REF *= 'reference'] inserts an ordered superscription and generates an auotomatic reference table at the end of the essay.
    • Added 6 essays from previous courseworks.
    • Remember to restart Apache server every time changes are pushed to the production. Otherwise, new changes will not be reflected.
    • Installed virtual environment and the lastest version of django locally. Previously the local django version is 1.7.1 while the production django version is 1.8.1. The version discrepancy causes the locally dumped data file can not be loaded by the production. The newer version removed the "name" field in contenttypes.contenttype while this field exists in the previous version.

    • Uploaded new post UIView + MHLayout. Updated HTML render engine to support rendering code snippets.

    • New design for Ideas front page. Every essay post includes title, post date, tags and an automatically extracted introduction from the essay body.
    • Now reference numbers in essay posts support hovering to view reference detail.
